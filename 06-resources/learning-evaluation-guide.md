# Guía de Evaluación del Aprendizaje

Esta guía define cuándo y cómo evaluar el conocimiento adquirido en cualquier tema. No se basa en tiempo real, sino en progreso temático.

---

## Estructura de Checkpoints

### Checkpoint 1: Fundamentos
**Cuándo:** Después de dominar 3-4 conceptos básicos del tema.

**Qué evaluar:**
- Sintaxis básica / terminología
- Conceptos atómicos (los más pequeños e indivisibles)
- Operaciones elementales

**Cómo evaluar:**
- Ejercicios guiados (tipo Exercism, LeetCode, Koans)
- Explicar el concepto en voz alta o por escrito sin consultar notas

---

### Checkpoint 2: Composición
**Cuándo:** Después de dominar cómo combinar los fundamentos.

**Qué evaluar:**
- Combinación de conceptos básicos
- Patrones comunes de uso
- Buenas prácticas iniciales

**Cómo evaluar:**
- Implementar algoritmos o soluciones que requieran múltiples conceptos
- Code katas o retos pequeños

---

### Checkpoint 3: Aplicación Práctica
**Cuándo:** Después de cubrir las herramientas principales del tema.

**Qué evaluar:**
- Uso de librerías/frameworks estándar
- Flujos de trabajo completos
- Manejo de casos edge

**Cómo evaluar:**
- Proyecto pequeño pero funcional
- Procesar datos reales (archivos, APIs, bases de datos)

---

### Checkpoint 4: Dominio
**Cuándo:** Después de cubrir temas avanzados o especializados.

**Qué evaluar:**
- Conceptos avanzados del tema
- Optimización y rendimiento
- Arquitectura y diseño

**Cómo evaluar:**
- Proyecto que integre todo lo aprendido
- Enseñar el tema a otra persona
- Contribuir a un proyecto open source

---

## Reglas Generales

| Señal | Acción |
|-------|--------|
| Terminas 3-4 temas relacionados | Mini-ejercicio práctico |
| Terminas un checkpoint completo | Proyecto pequeño que integre todo |
| 2+ semanas sin practicar el tema | Revisión rápida o re-hacer ejercicios anteriores |
| No puedes explicarlo sin notas | Aún no lo dominas, seguir practicando |

---

## Indicadores de Dominio

Has dominado un tema cuando puedes:

1. **Explicarlo** sin consultar notas
2. **Usarlo** en un contexto nuevo sin copiar código anterior
3. **Depurarlo** cuando algo falla
4. **Enseñarlo** a alguien más

---

## Aplicación por Área

### Lenguajes de Programación
| Checkpoint | Temas Típicos |
|------------|---------------|
| 1. Fundamentos | Variables, tipos, operadores, control de flujo |
| 2. Composición | Funciones, módulos, estructuras de datos |
| 3. Aplicación | Librerías estándar, I/O, manejo de errores |
| 4. Dominio | Concurrencia, metaprogramación, optimización |

### Frameworks
| Checkpoint | Temas Típicos |
|------------|---------------|
| 1. Fundamentos | Instalación, estructura de proyecto, conceptos core |
| 2. Composición | Rutas, controladores, modelos, vistas |
| 3. Aplicación | Autenticación, APIs, testing |
| 4. Dominio | Despliegue, escalabilidad, personalización avanzada |

### Conceptos (ej. patrones de diseño, arquitectura)
| Checkpoint | Temas Típicos |
|------------|---------------|
| 1. Fundamentos | Definición, problema que resuelve, cuándo usarlo |
| 2. Composición | Implementación básica, variantes |
| 3. Aplicación | Uso en proyecto real, combinación con otros patrones |
| 4. Dominio | Trade-offs, anti-patrones, cuándo NO usarlo |
